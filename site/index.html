// Features

先把日常工作流收进一个窗口

Conductor 不替你写提示词，也不包装成聊天产品。它解决的是更基础的问题：多个 agent 和终端同时跑时，状态、会话、成本和上下文别散在一堆窗口里。

01 Workspace

工作区和会话放在左侧

按目录组织工作区，当前 pane、最近会话和续聊入口都能找到。换项目时不用在终端窗口之间猜。

02 Panes

多 pane 真实终端

Codex、Claude Code 和普通 shell 可以并排跑。你随时能切进任意 pane，继续输入命令或接管上下文。

03 Usage

Token 和成本看得到

右侧用量面板按时间、项目和模型拆开统计。一次探索花了多少 token、主要花在哪个目录，一眼能查。

04 Queue

任务可以排队

给当前 pane 预先排好下一条指令，上一轮结束后继续执行。适合一批小修复、检查和清理任务。

05 Attention

待处理状态别漏掉

权限确认、提问、完成未读这类状态会集中提示。你不需要轮流点开每个终端看它是不是卡住了。

06 Native

原生 macOS 终端体验

ghostty 内核、Metal 渲染、分屏和中文输入都走原生应用路线。目标不是做一个网页聊天壳，而是让终端工作流更好管理。
